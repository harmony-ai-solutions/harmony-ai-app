# Phase 1-1: Jest 30 Upgrade & Preset Migration

## Objective

Modernize the test runner to Jest 30 and switch to the official `@react-native/jest-preset` package (extracted from core RN in 0.85). Jest 30 delivers ~37% faster test runs and ~77% lower peak memory vs Jest 29, plus ESM improvements and a new resolver. The new preset is the canonical path forward — the old `'react-native'` preset still works but is deprecated.

## Context

- **Why now**: The team is about to layer significant new test infrastructure (DB adapter, migration tests, integration tests). Modernizing the runner first avoids compounding risk later.
- **Why not Vitest**: See the research report — `vitest-native` is beta (~85% pass rate on `react-native-paper`), no public production RN migration stories as of mid-2026. Re-evaluate in ~12 months.
- **Reference**: [Jest 30 release blog](https://jestjs.io/blog/2025/06/04/jest-30), [RN 0.85 blog](https://reactnative.dev/blog/2026/04/07/react-native-0.85) (preset extraction), [Jest 30 migration guide](https://jestjs.io/docs/upgrading-to-jest30).

## Prerequisites

- Node.js ≥ 20.19.4 (already required by RN 0.86 — `engines.node` is `>=20` in `package.json`).
- A clean working tree (no in-flight changes to test files).

## Implementation Steps

### 1. Audit current Jest-related dependencies

Before changing versions, record what's installed:

```bash
npm ls jest @types/jest babel-jest @jest/globals react-test-renderer @types/react-test-renderer
```

Note any unexpected transitive versions.

### 2. Upgrade Jest packages

```bash
npm install --save-dev \
  jest@^30 \
  @types/jest@^30 \
  babel-jest@^30 \
  @jest/globals@^30
```

### 3. Install the new RN preset package

```bash
npm install --save-dev --save-exact @react-native/jest-preset@^0.86.0
```

The version should match the installed `react-native` major.minor.

### 4. Update `jest.config.js`

```diff
 module.exports = {
-  preset: 'react-native',
+  preset: '@react-native/jest-preset',
   transformIgnorePatterns: [
     'node_modules/(?!(react-native|@react-native|@react-navigation|react-native-fs|react-native-keychain|react-native-sqlite-storage|@react-native-documents|react-native-vector-icons|react-native-linear-gradient|react-native-safe-area-context|react-native-screens|react-native-paper)/)',
   ],
   setupFiles: ['./jest.setup.js'],
 };
```

> **Note**: The `transformIgnorePatterns` list may be simplified once `@react-native/jest-preset` is in place — it ships a sensible default. Try removing the override entirely first; only restore it if specific packages fail to transform.

### 5. Run the Jest 30 codemods / fixers

Jest 30 removed deprecated matcher aliases. The `eslint-plugin-jest` `no-alias-methods` rule has an autofixer:

```bash
npx eslint --fix \
  --rule '{"no-alias-methods": "error"}' \
  --parser-options=ecmaVersion:2022 \
  '__tests__/**/*.ts' '__tests__/**/*.tsx' 'src/**/*.test.ts' 'src/**/*.test.tsx'
```

Common rewrites:
- `toBeCalled()` → `toHaveBeenCalled()`
- `toThrowError()` → `toThrow()`
- `lastCalledWith()` → `toHaveBeenLastCalledWith()`

### 6. Rename `--testPathPattern` in any npm scripts

Jest 30 renamed this CLI flag. None currently exist in `package.json` (`"test": "jest"`), but check CI workflows and any docs.

### 7. Replace `jest.genMockFromModule` (if used)

Jest 30 removed this in favor of `jest.createMockFromModule`. Grep the repo:

```bash
# Use ripgrep via the Grep tool — search for "genMockFromModule" across the repo.
```

### 8. Update snapshots

Jest 30 changed snapshot formatting for some edge cases (React empty strings, Error causes). Run with `--updateSnapshot` and review the diff:

```bash
npx jest --updateSnapshot
git diff -- '**/*.snap'
```

Currently the repo has very few snapshots (none in `__tests__/`), so this should be a no-op.

### 9. Verify the existing suite still passes

```bash
npm test
```

Expected: `__tests__/App.test.tsx`, `__tests__/services/SyncService.test.ts` all pass. The `src/database/__tests__/*.test.ts` files do NOT currently run under Jest (they require the in-app runner) — that's expected and will be fixed in Phase 3.

### 10. (Optional) Add `@testing-library/react-native` for new tests

`react-test-renderer` is officially deprecated in React 19. The repo still depends on it (`react-test-renderer ^19.2.7` + `@types/react-test-renderer ^19.1.0` in `package.json`). For new component tests written as part of this overhaul, use RNTL instead:

```bash
npm install --save-dev @testing-library/react-native @testing-library/jest-native
```

Migrating existing snapshot tests off `react-test-renderer` is out of scope for this phase — defer to Phase 8 backlog.

## Files to Modify

- `package.json` — version bumps + new dev dependencies
- `package-lock.json` — regenerated by npm
- `jest.config.js` — preset swap (and possibly remove `transformIgnorePatterns` override)
- `jest.setup.js` — likely no changes; verify it still loads
- Any test files using deprecated matcher aliases (autofixed in step 5)

## Validation

- [ ] `npm ls jest` reports `jest@30.x.x`
- [ ] `npm test` exits 0 with no deprecation warnings
- [ ] No remaining `genMockFromModule` usages in repo
- [ ] `git diff jest.config.js` shows only the preset swap (plus optional transformIgnorePatterns removal)
- [ ] CI (after Phase 7) is green on a PR containing only these changes

## Rollback Plan

If Jest 30 introduces blockers, revert via:

```bash
git revert <commit>
npm install
```

The version bumps in `package.json` are isolated; reverting the commit and reinstalling restores the prior state.
